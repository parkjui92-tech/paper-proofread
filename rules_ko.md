# 한국어 학술 논문 교정 규칙 레퍼런스

이 파일은 Step 1(표기 교정)과 Step 2(문장 교열)에서 참조하는 규칙 모음이다. 각 항목은 패턴 → 수정안 → 근거 형식으로 정리되어 있다.

---

## 1. 빈출 맞춤법 오류

### 1.1 자주 틀리는 표기

| 틀린 표기 | 바른 표기 | 비고 |
|-----------|-----------|------|
| 데이타 | 데이터 | 외래어 표기법 |
| 컨텐츠 | 콘텐츠 | 외래어 표기법 |
| 메세지 | 메시지 | 외래어 표기법 |
| 억세스 | 액세스 | 외래어 표기법 |
| 프레임웍 | 프레임워크 | 외래어 표기법 |
| 갯수 | 개수 | 사이시옷 오적용 |
| 잇점 | 이점 | 발음 혼동 |
| 몇일 | 며칠 | 맞춤법 |
| 어의없다 | 어이없다 | 맞춤법 |
| 왠지 | 웬지 (← 상황에 따라 다름) | 왠지(왜인지) vs 웬(어찌 된) |
| 않되다 | 안 되다 | '않-'은 '~하지 않다'의 축약 |
| 되어지다 | 되다 | 이중피동 |
| 시켜지다 | 되다 | 이중피동 |
| 할께요 | 할게요 | 맞춤법 (ㄹ게) |
| 댓가 | 대가 | 사이시옷 오적용 |
| 보여지다 | 보이다 | 이중피동 |
| 만들어지다 | 만들다/만들어 내다 | 맥락에 따라 선택 |

### 1.2 띄어쓰기 핵심 규칙

#### 의존명사 (반드시 띄어 씀)

| 패턴 | 올바른 표기 | 예시 |
|------|-------------|------|
| ~ㄹ 수 | ~할 수 있다 | ✕ 할수있다 |
| ~ㄴ 바 | ~한 바 있다 | ✕ 한바 있다 |
| ~ㄴ 것 | ~한 것이다 | ✕ 한것이다 |
| ~ㄹ 때 | ~할 때 | ✕ 할때 |
| ~ㄴ 데 | ~하는 데 (장소/상황) | ✕ 하는데 (단, 접속어미 '-는데'는 붙임) |
| ~ㄹ 뿐 | ~할 뿐이다 | ✕ 할뿐이다 |
| ~ㄴ 지 | ~한 지 (시간 경과) | ✕ 한지 |
| ~ㄹ 만큼 | ~할 만큼 | ✕ 할만큼 |

#### 보조용언 (띄어 쓰는 것이 원칙, 붙여 쓰는 것도 허용 — 단, 논문 내 통일 필수)

| 패턴 | 원칙 (띄어쓰기) | 허용 (붙여쓰기) |
|------|-----------------|-----------------|
| ~해 보다 | 시도해 보다 | 시도해보다 |
| ~해 주다 | 설명해 주다 | 설명해주다 |
| ~해 오다 | 연구해 오다 | 연구해오다 |
| ~해 나가다 | 추진해 나가다 | 추진해나가다 |
| ~해 내다 | 이루어 내다 | 이루어내다 |
| ~해 버리다 | 사라져 버리다 | 사라져버리다 |

**교정 원칙**: 둘 다 맞는 경우, 논문 전체에서 하나의 방식으로 통일되어 있는지 확인. 통일되어 있으면 수정하지 않음. 혼용되어 있으면 다수 용례 쪽으로 통일 권고.

### 1.3 문장부호

| 규칙 | 설명 | 예시 |
|------|------|------|
| 괄호 안 마침표 | 괄호가 문장 일부이면 마침표를 괄호 밖에 | ~이다(표 1 참조). |
| 가운뎃점 | 대등한 두 요소 병렬 | 과학·기술, 수도권·비수도권 |
| 쉼표 사용 | 나열, 부속 성분 구분 | 첫째, ~ 둘째, ~ |
| 쌍점 | 부연 설명 도입 | 다음과 같다: |
| 따옴표 | 인용 시 큰따옴표, 강조/정의 시 작은따옴표 | '기술시장'이란 ~ |

## 2. 번역투 교정 패턴

학술 논문에서 자주 발견되는 번역투와 그 교정안이다. 심각도는 대부분 🟡 suggest이다.

| 번역투 패턴 | 교정안 | 비고 |
|-------------|--------|------|
| ~에 있어서 | ~에서 / 삭제 | "연구에 있어서" → "연구에서" |
| ~하는 것이 가능하다 | ~할 수 있다 | 영어 "it is possible to" 직역 |
| ~라고 하는 | ~라는 | 불필요한 확장 |
| ~하는 것이 필요하다 | ~해야 한다 / ~할 필요가 있다 | 영어 "it is necessary to" 직역 |
| ~것으로 사료된다 | ~(으)로 판단된다 / ~(으)로 보인다 | 과도한 겸양 |
| ~것으로 보여진다 | ~(으)로 보인다 | 이중피동 + 번역투 |
| ~에 기인한다 | ~때문이다 / ~에서 비롯된다 | 한자어 직역투 |
| ~를 통하여 | ~(으)로 / ~을 거쳐 | 맥락에 따라 선택 |
| ~에 대한 ~에 대한 (이중) | 하나를 다른 조사로 대체 | 조사 중복 |
| ~의 ~의 ~의 (연쇄) | 관형절로 풀기 또는 조사 변경 | 과도한 'の' 구조 |
| ~함으로써 ~할 수 있다 | ~하면 ~할 수 있다 | 간결화 |
| ~가 행해졌다 | ~가 이루어졌다 / ~를 수행했다 | 일본어투 |
| ~에 의해 ~되다 | 능동문으로 전환 우선 검토 | 영어 수동태 직역 |
| ~적(的) 남용 | 필요한 경우만 사용 | "효율적", "체계적"은 OK. "본질적인 기본적인 핵심적인" 연속은 과다 |

## 3. 학술 글쓰기 관용 표현 & 권장 표현

### 3.1 부적절한 표현 → 권장 표현

| 부적절 | 권장 | 사유 |
|--------|------|------|
| 아주/매우/굉장히 | 상당히 / 현저하게 / 유의미하게 | 학술적 객관성 (단, 과도한 한자어도 지양) |
| ~라고 생각한다 | ~(으)로 판단된다 / ~(으)로 분석된다 | 주관 표현 지양 |
| 요즘/최근 들어 | 최근 / 근래 | 구어체 지양 |
| ~하고 있는 중이다 | ~하고 있다 | 중복 표현 |
| ~라는 사실을 알 수 있다 | ~(으)로 나타났다 / ~임을 확인했다 | 불필요한 확장 |
| 이런/저런/그런 | 이러한/그러한 / 해당 | 구어체 지양 |
| 되게/엄청 | 삭제 또는 대체 | 구어체 |

### 3.2 단수/복수 명확화 (한국어에서 주의)

- "연구자들은" vs "연구자는": 맥락에 따라 선택하되 논문 내 통일
- "결과들" 같은 불필요한 복수 접미사 '-들' 남용 지양

### 3.3 능동/피동 가이드

학술 논문에서 피동태 자체는 문제가 아니지만, 아래 경우에는 능동으로 전환을 권고한다:
- 행위 주체가 명확한데 피동태를 쓴 경우: "분석이 수행되었다" → "본 연구에서는 ~를 분석했다"
- 피동태가 3문장 이상 연속되는 경우: 한 문장은 능동으로 전환
- 이중피동: 절대 사용 금지 (error 등급)

## 4. 인용·참고문헌 규칙

### 4.1 APA 7th 기본 형식

**본문 인용:**
- 저자 2인: (김영희 & 이철수, 2023) 또는 김영희와 이철수(2023)
- 저자 3인 이상: (김영희 외, 2023) 또는 김영희 외(2023)
- 영문: (Smith et al., 2023)
- 직접 인용: (김영희, 2023, p. 45)

**참고문헌 목록:**
- 학술지: 저자. (연도). 제목. *학술지명*, *권*(호), 시작페이지-끝페이지. DOI
- 단행본: 저자. (연도). *제목*. 출판사.
- 보고서: 기관명. (연도). *보고서명* (보고서 번호). URL
- 정렬: 한글 가나다순 → 영문 알파벳순 (또는 학회 규정에 따름)

### 4.2 체크리스트

- [ ] 본문 인용 건수 = 참고문헌 목록 건수 (누락·잉여 없음)
- [ ] 연도 일치 (본문 2023 ↔ 목록 2023)
- [ ] 저자명 일치 (본문 약칭과 목록 풀네임 대응)
- [ ] 학술지명 이탤릭 처리 (해당 스타일 요구 시)
- [ ] 페이지 범위 형식 통일 (pp. 1-20 vs 1–20)
- [ ] URL/DOI 형식 통일

## 5. 심각도 분류 기준

| 심각도 | 태그 | 기준 | 예시 |
|--------|------|------|------|
| 🔴 error | 반드시 수정 | 규범 위반, 사실 오류, 인용 불일치 | 맞춤법 오류, 없는 문헌 인용 |
| 🟡 suggest | 수정 권고 | 가독성·명확성 저하 | 번역투, 문장 과다 복잡 |
| 🔵 style | 선택적 | 스타일 선호 차이 | 문장 길이 분리, 접속사 변경 |

aggressive 모드에서는 🔵 style도 적극 제시하고, conservative 모드에서는 🔴 error만 보고한다.
